# E2E Plan: login

> 生成日: 2026-06-25 / 入力源: 実サイト探索(chrome-devtools) + seed test
> 対象URL: http://localhost:3000/login
> 認証: 要 / 方式=form(email+password) / ロール=未ログイン・user・admin / 探索=user で手動ログイン済みセッションに乗って実施

## 遷移マップ

### 前提データ / 環境
- **認証戦略**: storageState 前提。`login.setup.ts` が user でログインし `e2e/.auth/user.json` を作る。
  各テストは state で開始し、ログイン操作は踏まない。**ログインフロー検証(S1〜S3,S7)だけ未ログイン開始**で書く。
- 必要なシードデータ: 有効ユーザー `user@example.com` / 管理者 `admin@example.com`
- 除外範囲: パスワードリセットのメール送信実体、外部IdP（SSO）

### 画面・遷移表

| # | 画面 | 到達操作 | 分岐条件 | 非同期イベント | 期待結果 | 確認状況 |
|---|------|----------|----------|----------------|----------|----------|
| 1 | ログイン画面 | `/login` へ遷移 | - | - | フォーム表示 | 確認済 |
| 2 | ダッシュボード | 有効な認証で submit | 認証成功 | `POST /api/login` 応答待ち | 一覧表示 | 確認済 |

### 未確認・要レビュー
- 管理者ロール時の追加メニュー（#5）は seed の admin で要確認
- ネットワーク遅延時のローディング表示の有無

## シナリオ仕様

### S1. ログイン成功（happy path）
- **開始状態**: 未ログイン / 有効ユーザー存在 / `/login`
- **操作**:
  1. メールアドレスに `user@example.com` を入力
  2. パスワードに `password` を入力
  3. 「ログイン」を押下
- **中間観測点**: 押下後にローディング表示 → `POST /api/login` 完了で `/dashboard` へ遷移
- **終了条件**: ダッシュボード見出しと「user@example.com」表示
- **除外事項**: パスワードリセット導線

### S2. 必須項目未入力（validation error）
- **開始状態**: 未ログイン / `/login`
- **操作**: 空のまま「ログイン」を押下
- **中間観測点**: `/api/login` は呼ばれない
- **終了条件**: 「メールアドレスを入力してください」が表示
- **除外事項**: -

### S3. 不正な認証（permission/認証失敗）
- **開始状態**: 未ログイン / `/login`
- **操作**: 不正なパスワードで submit
- **中間観測点**: `POST /api/login` が 401 を返す
- **終了条件**: 「メールアドレスまたはパスワードが違います」表示、`/login` に留まる
- **除外事項**: アカウントロックの回数制御

### S4. 戻る操作
- **開始状態**: storageState=user で `/dashboard`（ログイン操作は踏まない）
- **操作**: ブラウザの戻るを押下
- **中間観測点**: -
- **終了条件**: 未ログインに戻らず `/dashboard` に留まる（または `/login` へ正しくリダイレクト）
- **除外事項**: -

### S5. 再読込
- **開始状態**: storageState=user で `/dashboard`（ログイン操作は踏まない）
- **操作**: ページをリロード
- **中間観測点**: セッション復元
- **終了条件**: ログイン状態が保持され `/dashboard` 表示
- **除外事項**: -

### S6. 途中離脱
- **開始状態**: `/login` でメールのみ入力
- **操作**: 別ページへ遷移し `/login` に戻る
- **中間観測点**: -
- **終了条件**: 入力途中値の保持仕様に従う（保持しないなら空に戻る）
- **除外事項**: -

### S7. ネットワーク遅延（happy path の遅延版）
- **開始状態**: 未ログイン / `/login` / `/api/login` を 3s 遅延にモック
- **操作**: 有効な認証で submit
- **中間観測点**: 応答までローディングインジケータが表示され続ける、ボタンは二重押下不可
- **終了条件**: 応答後 `/dashboard` へ遷移
- **除外事項**: タイムアウト時のリトライUI

### S8. 権限差分（permission / user に管理メニュー非表示）
- **開始状態**: storageState=user で `/dashboard`
- **操作**: ダッシュボードのナビゲーションを確認
- **中間観測点**: ユーザー名 `user@example.com` が表示される
- **終了条件**: 「管理メニュー」リンクが**表示されない**（admin 側は `e2e/.auth/admin.json` を作り、別 project=storageState=admin で「表示される」対を検証）
- **除外事項**: 管理機能の各操作の中身
