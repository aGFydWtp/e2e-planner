# E2E Plan: board

> 生成日: 2026-06-26 / 入力源: PRD + 実サイト探索(chrome-devtools)
> 対象URL: https://app.example.com/board
> 認証: 要 / 方式=SSO（外部IdP=Google Workspace）/ ロール=未ログイン・member・admin / 探索=member で手動ログイン済みセッションに connectOverCDP で乗って実施
> 探索環境: ロケール=ja-JP / タイムゾーン=Asia/Tokyo

## 遷移マップ

### 前提データ / 環境
- **中心価値（PRD より）**: チームメンバーがプロジェクトのタスクボードでタスクを「作成→完了→削除」してプロジェクト進行を管理する。
- **認証戦略**: SSO のため form 自動ログイン不可。storageState は実ログイン済みブラウザから connectOverCDP で採取する前提。各テストは state で開始する。
- 必要なシードデータ: member `member@example.com` / admin `admin@example.com` / 既存プロジェクト「Demoプロジェクト」
- 除外範囲: 外部IdP（Google）のログイン画面自体の自動化、課金、Slack通知連携

### 状態グラフ（タスク作成フォーム = 段階UI）
- ボード → 「新規タスク」押下 → 入力フォーム展開（段階1: タイトル入力中）→ 保存 → 段階2: 一覧に反映
- member には #6「招待」導線は非表示 / admin には表示（permission差分の素地）
