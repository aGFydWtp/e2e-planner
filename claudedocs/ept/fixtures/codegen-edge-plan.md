# E2E Plan: board  ✅ 承認済み（承認ゲート①通過・Step3 codegen 入力）

> 生成日: 2026-06-26 / 入力源: PRD + 実サイト探索(chrome-devtools)
> 対象URL: https://app.example.com/board
> 認証: 要 / 方式=SSO（外部IdP=Google Workspace）/ ロール=未ログイン・member・admin
> 探索環境: ロケール=ja-JP / タイムゾーン=Asia/Tokyo

## 認証戦略（確定）
- SSO のため form 自動ログインは不可。storageState は実ログイン済みブラウザから採取済みの前提:
  `e2e/.auth/member.json`（member）/ `e2e/.auth/admin.json`（admin）。各テストは state で開始する。
- **ログイン成功 / 検証エラー / 認証失敗(401) は該当なし（SSO のためログインフロー自体は検証対象外）。**

## 破壊的シナリオの決定（一括提示の結果・承認済み）

| 破壊的シナリオ | 破壊性 | teardown 可否 | 既定 | **決定（確定）** |
|----------------|--------|---------------|------|------------------|
| タスク作成→完了→削除（自己完結1本） | 作成/更新/削除 | 可（UI で削除） | 自己完結 | **自己完結（承認済み）** |
| メンバー招待（招待メール送信） | メール送信 | 不可（外部副作用） | 要慎重 | **除外（不可逆・ユーザー確認の上で除外）** |

## UI 上の注意（探索で観測）
- `/board` 初回表示時、画面上部に「新機能のお知らせ」**お知らせバナー**（×で閉じられる）が出ることがある。出ないこともある。検証本体には干渉しない。

## シナリオ仕様

### S1. ボード閲覧（happy path・非破壊）
- **開始状態**: storageState=member で `/board`
- **操作**: ボードを開く
- **中間観測点**: `GET /api/tasks` 応答までスケルトン表示
- **終了条件**: カンバン見出し「Demoプロジェクト」と既存タスクが表示される
- **後始末**: - / **除外事項**: タスク詳細パネルの中身

### S2. 権限差分（permission）
- **開始状態（a）**: 未ログイン（storageState なし）で `/board` にアクセス
  - **終了条件（a）**: 外部IdP のログイン画面へリダイレクト（`accounts.google.com` 等の外部 origin へ遷移、または `/login` 経由）
- **開始状態（b）**: storageState=member で `/board` / storageState=admin で `/board`
  - **終了条件（b）**: member には「招待」ボタンが**非表示**、admin には**表示**される
- **後始末**: - / **除外事項**: 招待後の挙動（招待自体は除外）

### S3. 再読込
- **開始状態**: storageState=member で `/board`
- **操作**: リロード
- **終了条件**: セッション保持、カンバンが再表示される
- **後始末**: - / **除外事項**: -

### S4. 戻る操作
- **開始状態**: storageState=member で `/board` → タスク詳細を開いた状態
- **操作**: ブラウザの戻る
- **終了条件**: ボード一覧に戻り、セッションは維持
- **後始末**: - / **除外事項**: -

### S5. 途中離脱
- **開始状態**: storageState=member で `/board`、「新規タスク」フォームにタイトルだけ入力
- **操作**: 別画面へ遷移し `/board` に戻る
- **終了条件**: 下書き保持仕様に従う（保持しないなら空のフォームに戻る）
- **後始末**: - / **除外事項**: -

### S6. ネットワーク遅延
- **開始状態**: storageState=member で `/board` / `GET /api/tasks` を 3s 遅延にモック
- **操作**: ボードを開く
- **中間観測点**: 応答までスケルトン/ローディングが表示され続ける
- **終了条件**: 応答後にカンバンが表示
- **後始末**: - / **除外事項**: 失敗時リトライUI

### S7. タスク作成→完了→削除（破壊的・**自己完結**・承認済み）
- **開始状態**: storageState=member で `/board`
- **操作**:
  1. 「新規タスク」を押下
  2. タイトルにユニーク名を入力（例: `e2e-task-<timestamp>`）
  3. 「保存」を押下 → 一覧に新規行が追加される
  4. 追加された当該タスク行のチェックボックスを ON にして「完了」へ移す
- **中間観測点**:
  - 保存で `POST /api/tasks` 完了後、**一覧の件数が1増える**
  - 完了操作で当該行が「完了」列へ移動
- **終了条件**: 作成したユニーク名のタスクが「完了」状態で表示される
- **後始末（teardown 必須）**: 作成したのと同じ UI 経路で当該タスクを削除する（行メニュー→削除→確認）。削除後、その名前のタスクが存在しないことを検証する。
- **除外事項**: 他人が作成したタスクには触れない

## 必須網羅クラスの充足メモ
- happy=S1 / permission=S2 / 戻る=S4 / 再読込=S3 / 途中離脱=S5 / 遅延=S6 / （破壊的価値フロー=S7）
- **validation error**: タスク作成のタイトル空 submit を S7 の派生で1件足してもよいが、本 plan では「該当（タイトル必須）」として codegen 側で S2 とは別に起こす余地あり。
- **ログイン成功/検証エラー/401**: 該当なし（SSO・上記認証戦略）
